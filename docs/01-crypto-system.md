# 01. 암호 시스템 (Crypto System)

## 통신 보안 개요

- 오늘날 인터넷은 통신과 정보 전달을 위해 가장 효과적이고 많이 사용되는 정보 전달 수단이다.
- Internet망을 통해 많은 사람들이 통신함에 따라 통신 보안은 매우 중요한 이슈 중 하나이다.
- 통신 보안은 모든 네트워크 설계의 중요한 요소이며, 데이터를 암호화하는 기법들은 현대 정보
  시스템에 없어서는 안 될 필수 요소가 되었고, 안전한 접근을 위해 **안정성·신뢰성·기밀성**을 제공한다.

## 암호 시스템 (CryptoSystem)

- 암호 시스템은 통신 보안을 제공하기 위해 암호화 응용 프로그램을 포함하는 하나의 프레임워크다.
- 암호화 프로토콜, 암호화 절차, 암호화 알고리즘 등을 포함한 개념으로, 암호화 기술을 사용하여
  인코딩과 디코딩하는 시스템을 운용하기 위한 기술이다.
- 정보의 **기밀성, 무결성**을 구현하기 위해 다음 기술을 이용한다.
  - 암호화 (Encryption)
  - 복호화 (Decryption)
  - 해쉬 함수 (Hash-function)
  - 키 관리 기법 (Key Management)

## 암호화 기법의 개요

- 암호화 기법은 오래 전부터 있어온 과학으로, 기원전 1900년경 이집트인들이 고대 비문을 만들 때
  사용했고, 로마인들은 비밀 메시지를 교환할 때 사용하였다.
- 암호 기법은 그리스어 **'Kryptos'(숨겨진)** 와 **'Graphein'(기록하다)** 에서 유래되었다.

## 주요 용어

| 용어 | 설명 |
|:---|:---|
| 평문 (Plaintext) | 암호화되지 않은 일반 데이터 |
| 암호화 (Encryption) | 비밀 Key로 일반 데이터를 비밀 코드로 전환하는 연산 프로세스. 의도된 수신자 외에는 정보 접근 불가 |
| 복호화 (Decryption) | 암호화의 반대 과정. 암호문을 원본 평문으로 변환 |
| 암호문 (Ciphertext) | 암호 프로세스를 거친 결과물 |
| 해시 (Hash, 인증) | Message-digest. Text 수열에 수학 공식을 적용해 산출한 유일한 값 (데이터 식별용) |

## 암호화 알고리즘 분류

```text
┌─────────────────────────────┬──────────────────────────────────────┐
│ 대칭 키(비밀키) 알고리즘       │ 하나의 Key로 암호화/복호화 (Key 동일)    │
│ Symmetric Key                │ EX: DES, 3DES, AES, SEED              │
├─────────────────────────────┼──────────────────────────────────────┤
│ 비대칭 키(공개키) 알고리즘     │ 두 개의 Key 사용 (암호화 Key ≠ 복호화 Key)│
│ Asymmetric Key               │ EX: RSA, Diffie-Hellman               │
├─────────────────────────────┼──────────────────────────────────────┤
│ 해시 알고리즘                 │ 단방향 함수, 원본 복원 불가             │
│ Hash Algorithm               │ EX: MD5, SHA                          │
└─────────────────────────────┴──────────────────────────────────────┘
```

---

## 1. 대칭 키 암호화 알고리즘 (비밀키)

- 하나의 Key로 암호화 및 복호화가 이루어지며, 기밀성 제공을 위해 메시지 내용을 암호화한다.
- 일반적으로 두 가지 방식으로 전송이 가능하다.

```text
- Streaming Cipher : 비 암호화 데이터를 bit 단위로 암호화
                     → CPU 소모율 증가, 전송 시 Delay 증가 가능

- Block Cipher     : 비 암호화 데이터를 고정된 길이의 Block 단위로 암호화
                     → 128bit 입력 시 128bit Block으로 출력 (DES, AES)
```

| 알고리즘 | Key 길이 | 특징 |
|:---|:---|:---|
| **DES** | 56bit | 1970년대 IBM 개발, 1976 FIPS 표준. 64bit 블록 암호화. 현재 12시간 내 해독 가능 |
| **3DES** | 168bit | DES 변형. 56bit Key를 3회 반복(암호화/복호화 3회). DES 임시 대체 |
| **AES** | 128 / 192 / 256bit | Rijndael. 2001 NIST 소개, 2002 표준. Block/Key 길이 조합 사용. 현재 가장 일반적 |

> 💡 DES: NIST가 5년마다 안전도 평가, 1993년 재평가에서 문제 판단 → 1998년까지만 표준 사용
> 💡 AES: 현재 컴퓨터로 해독 시 약 148조 년 소요로 발표됨

---

## 2. 비대칭 키 암호화 알고리즘 (공개키)

- 1976년 처음 공개. 두 개의 Key 사용 (평문 암호화 Key + 암호문 복호화 Key).
- 대칭 키와 달리 **비밀키 공유 없이** 보안이 유지되지 않는 채널을 통해 보안 구현 가능.
- 일반적으로 디지털 인증이나 Key 관리에 사용된다.
  (데이터 암호화 전송도 가능하나, 장비 부하로 인해 대칭 키 방식이 더 효율적)

| 알고리즘 | 설명 |
|:---|:---|
| **RSA** | 1976년 MIT의 Rivest·Shamir·Adleman 개발. Key교환, 전자서명, 메시지 암호화에 사용 |
| **Diffie-Hellman** | 1976년 Martin Hellman·Whitfield Diffie 발표. Key Exchange Protocol. 안전하지 않은 채널에서 **비밀키 교환 전용** (전자서명 ✕) |

---

## 3. 해시 알고리즘 (Hash Algorithm)

- Hash function, Message-digest 등으로도 불린다.
- 수학적 공식으로 원본 Text 기반 고정 길이 해쉬값을 계산하며, 해쉬값을 원본 데이터로 **복원할 수 없다.** (one-way 알고리즘)
- 종류: **MD5** (RFC 1321), **SHA / SHA-1** (RFC 3174)
